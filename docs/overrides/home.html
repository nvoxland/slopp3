An agent-native Clojure codebase. Your code lives in a store, not in .clj files. Every write is compile-gated, test-verified and stamped with the intent that produced it, against a live JVM image that has been running your program the whole time.

what you get

slopp is one bet carried all the way down: make the top-level form the atom of editing, storage, hot-reload, verification and history, and the authoring loop gets both faster and harder to fool.

No files to edit

Namespaces are ordered forms in a SQLite delta journal. A VFS renders .clj text on demand; build materializes files only when outside tooling needs them.

Address code by name

Namespace plus form name. Reads return one form with interface cards for everything it calls. Nothing on the wire ever carries a line number.

Verified before it lands

Dialect gate, compile into the live image, cold-load check, lint refusal, then the tests that exercise the touched forms. A failed gate commits nothing.

The image is the oracle

A persistent JVM runs your code. Behaviour questions get answered by running it -- query_eval, query_call, query_observe -- instead of by reading callers.

Intent is stored, not guessed

Every delta carries the agent, its stated reason, and the user's verbatim ask. Replay a form's life, time-travel to any point, or find the dead ends someone already walked.

Architecture that is enforced

Modules with declared edges, package-private deep namespaces, and purity tiers that hard-refuse an effect reaching into a pure core.

Merges at form grain

Branches are O(1) snapshots and merging is delta replay. Different-form work lands, identical changes converge, same-form divergence is a real conflict.

Still a git repo

Milestones project into ordinary git commits on a branch slopp owns. Push, pull, review PRs; edits made on GitHub come back through a form-granular 3-way merge.

the loop

A whole task is usually 10-20 tool calls. The mechanical parts -- turns, identity, formatting, ordering, test selection -- are the system's job, not yours.

session_brief One small call: namespaces, form names, recent milestones and what was asked for.
query_slice The form you are about to change, plus cards for everything it reaches.
edit_* Small writes, REPL style. Each one verifies itself and reports.
done At every point you think a piece of work is finished. Tidies, lints, re-verifies, reports.
commit_point The milestone a human diffs and reverts to -- green-gated, and a git commit.

slopp is experimental and self-hosted: it has been developed through itself since the store flip, and its published repo is a projection of its own store. Expect sharp edges.
